# 4.2｜反向传播：梯度如何穿过网络

## 1. 计算图

前向传播把简单操作连成图：矩阵乘法、加法、激活和损失。反向传播从损失出发，沿相反方向应用链式法则，为每个参数计算梯度。

以输出层为例，sigmoid 与二元交叉熵合并后，对 logit 的梯度简化为：

```text
dL/dz = (p-y)/n
dL/dW₂ = hᵀ(dL/dz)
dL/db₂ = sum(dL/dz)
```

梯度传入隐藏层：

```text
dL/dh = (dL/dz)W₂ᵀ
d(tanh(a))/da = 1-tanh(a)²
dL/da = dL/dh × (1-h²)
dL/dW₁ = Xᵀ(dL/da)
```

这里 `×` 是逐元素乘法。矩阵形状既是实现指南，也是检查推导的工具。

## 2. 更新顺序

必须先计算完所有梯度，再更新所有参数。如果先更新 `W₂`，随后计算隐藏层梯度时就混入了新参数，得到的不是同一次前向传播对应的梯度。

## 3. 梯度消失与爆炸

反向传播会连乘局部导数。许多小于 1 的导数相乘导致梯度消失，许多大值相乘导致爆炸。ReLU、合理初始化、残差连接、归一化和梯度裁剪分别从不同角度缓解问题。

## 4. 初始化

所有权重都初始化为零会让同一层神经元保持完全对称，学到相同特征。权重应随机初始化，并按输入维度缩放；偏置通常可从零开始。

## 5. 自动微分不是魔法

PyTorch 会记录张量操作，`loss.backward()` 自动按链式法则累积梯度。你仍需负责：损失是否正确、shape 是否正确、是否清零旧梯度、是否在验证阶段关闭梯度。

## 6. 用数值梯度检查

选择一个参数，加减很小的 ε，比较损失变化得到的数值梯度与反向传播梯度。相对误差很小时实现更可信。不要用过小 ε，否则浮点舍入误差会变大。

## 7. 阅读代码的顺序

打开 [NumPy MLP](../../scripts/numpy_mlp.py)，依次找：参数初始化 → `forward` → 损失 → 输出层梯度 → 隐藏层梯度 → 参数更新。逐行写出 shape。
