# 第一课｜机器学习到底在做什么

## 1. 一句话直觉

机器学习是让程序从样本中总结规律，再对未见过的数据作出预测。传统编程常由人写规则；机器学习则由人定义数据、目标、模型范围和评价方式，让算法从数据中找到参数。

## 2. 六个核心词

- **样本**：一条观测，例如一套房子。
- **特征**：用于预测的信息，例如面积、位置和房龄。
- **标签**：希望预测的答案，例如成交价。
- **模型**：把特征映射到预测结果的函数。
- **损失**：衡量单次或整体预测有多差的数值。
- **泛化**：模型在未见过的数据上仍然有效的能力。

## 3. 三类常见任务

| 任务 | 输出 | 例子 |
|---|---|---|
| 回归 | 连续数值 | 房价、温度、销量 |
| 分类 | 离散类别 | 垃圾邮件、疾病类别、图片类别 |
| 聚类 | 数据中的群组 | 客户分群、文档主题探索 |

## 4. 为什么要拆分数据

训练集用于学习参数；验证集用于选择模型和超参数；测试集只在最终评估时使用。如果你反复查看测试集并据此改模型，测试集就间接参与了训练，最终分数会过于乐观，这叫数据泄漏的一种形式。

## 5. 第一个模型：一条直线

假设用学习小时数 `x` 预测分数 `y`：

```text
预测分数 = 权重 × 学习小时数 + 偏置
ŷ = w × x + b
```

训练的目标就是找到合适的 `w` 和 `b`。均方误差衡量预测与真实值的平均平方差；梯度下降沿着让损失下降的方向逐步更新参数。

运行示例：

```powershell
python scripts/linear_regression_from_scratch.py
```

## 6. 练习

1. 预测明天是否下雨属于哪类任务？为什么？
2. 预测明天降雨量属于哪类任务？
3. 如果训练准确率 100%，测试准确率 60%，最可能发生了什么？
4. 修改示例中的学习率为 `1.0`，观察损失并解释结果。
5. 加入一个明显错误的数据点，观察参数如何变化。

## 7. 自测答案

1. 二分类，因为答案是“下雨/不下雨”。
2. 回归，因为输出是连续数值。
3. 常见原因是过拟合，也要排查训练/测试分布差异和预处理错误。
4. 学习率过大可能越过最低点，使损失振荡甚至发散。
5. 均方误差会放大离群点影响，直线会被明显拉动。
