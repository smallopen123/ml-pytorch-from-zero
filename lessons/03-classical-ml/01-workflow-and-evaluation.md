# 3.1｜正确的机器学习流程与评价

## 1. 先定义问题，再选算法

明确一行数据代表什么、预测时能获得哪些特征、标签如何产生、错误的业务代价是什么。预测“是否违约”是分类，但如果标签在放款后一年才知道，训练样本和上线样本的时间边界必须一致。

## 2. 三份数据的职责

- 训练集：拟合参数。
- 验证集：选择算法、特征、阈值和超参数。
- 测试集：所有选择结束后，只做一次最终评估。

小数据常用 K 折交叉验证：轮流用 1 折验证、其余训练，最后汇总。测试集仍应独立保留。

## 3. 数据泄漏

以下操作如果在拆分前完成，都会泄漏：

- 用全数据计算均值、标准差或缺失值填充值。
- 用全数据选择与标签最相关的特征。
- 同一用户、病人或时间段的数据同时出现在训练和测试中。
- 将预测时不可能知道的未来信息作为特征。

scikit-learn 的 `Pipeline` 能让预处理在每个训练折内部拟合，是防泄漏的重要工具。

## 4. 指标不是越多越好

回归常用 MAE、RMSE、R²。分类常用准确率、精确率、召回率、F1、ROC-AUC、PR-AUC。类别极不平衡时，准确率容易误导。

```text
precision = TP / (TP + FP)
recall    = TP / (TP + FN)
F1        = 2PR / (P + R)
```

垃圾邮件拦截可能重视精确率，癌症初筛通常更重视召回率。指标选择必须对应错误成本。

## 5. 永远先做基线

回归可用训练集均值，分类可用多数类或简单逻辑回归。复杂模型只有稳定超过基线才有价值。记录随机种子、数据版本、特征、参数和指标。

## 6. 练习

1. 设计一个“房价预测”的数据拆分方案。
2. 为什么先标准化全数据再交叉验证会泄漏？
3. 1000 个样本中只有 10 个阳性，全部预测阴性的准确率是多少？它有用吗？
4. 为垃圾邮件和疾病筛查分别选择主要指标并解释。
