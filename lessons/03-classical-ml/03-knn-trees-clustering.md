# 3.3｜KNN、树模型与 K-Means

## 1. KNN：让邻居投票

KNN 不显式训练参数。预测时计算新样本与训练样本的距离，取最近的 K 个邻居投票。K 小时边界灵活、容易过拟合；K 大时更平滑、可能欠拟合。

优点：直观、适合小数据基线。缺点：预测慢、对尺度和无关特征敏感、高维中距离区分度下降。

[查看 KNN NumPy 实现](../../scripts/classical_ml_from_scratch.py)。

## 2. 决策树：连续提出问题

决策树选择特征和阈值切分数据，让子节点更“纯”。分类常用 Gini 或熵；回归常用均方误差。树无需标准化、能表达非线性和交互，但深树容易记住训练数据。

随机森林对多棵随机化树投票，降低方差；梯度提升按顺序训练弱树，每棵树纠正之前的错误。表格数据中树集成通常是强基线。

关键超参数：最大深度、叶节点最少样本数、树数量、学习率。所有选择必须只看验证集/交叉验证。

## 3. K-Means：没有标签时找中心

K-Means 重复两步：

1. 将每个样本分给最近中心。
2. 将中心更新为簇内样本均值。

目标是最小化样本到所属中心的平方距离。它适合近似球形、尺度相近的簇；对离群点、初始中心和特征尺度敏感。

[查看 K-Means NumPy 实现](../../scripts/classical_ml_from_scratch.py)。

## 4. 监督与无监督的区别

KNN/树使用标签并根据预测错误学习；K-Means 没有标签，只按几何结构分组。聚类编号没有固有含义，簇 0 不比簇 1“更低”。聚类结果必须结合业务或外部标签解释。
